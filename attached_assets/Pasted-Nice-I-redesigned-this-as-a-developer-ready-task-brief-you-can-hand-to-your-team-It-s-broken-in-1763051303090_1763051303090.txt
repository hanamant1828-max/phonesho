# POS UI ‚Äî Professional redesign: developer task brief

---

## Objective

Make the POS screen look modern, professional, and easy to use on desktop and tablet. Keep the same features (Sale / Return / Exchange, search, cart, customer info, bill summary) but improve layout, visual hierarchy, spacing, and interactions.

---

## Priorities (high ‚Üí low)

1. Visual hierarchy & spacing (typography, margins, grid)
2. Clean, prominent search field & product results
3. Compact, clear cart area with quick actions
4. Right-side collapsible order summary (customer info + bill)
5. Responsive behavior and keyboard/scanner flow
6. Accessibility & performance
7. Polishing: icons, micro-interactions, animations

---

## Global design system (to be shared as tokens)

* *Font:* Inter or Poppins (system fallback: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue")
* *Primary color:* #1E6FFF (blue)
* *Accent / success:* #10B981 (green)
* *Neutral text:* #111827 (dark) / #6B7280 (muted)
* *Surface / card:* #ffffff; subtle border #E6E9EF
* *Shadow:* 0 6px 20px rgba(16,24,40,0.06)
* *Border radius:* 8px for inputs/cards, 12px for major containers
* *Spacing scale:* 4px base (4,8,12,16,24,32,48)

---

## Layout & navigation

* Use a two-column layout:

  * Left (main): 70% ‚Äî search + product list or cart list
  * Right (aside): 30% ‚Äî collapsible Customer Info + Bill Summary
* Top: compact app header with logo + page title + back button + action buttons (Clear Cart, Hold, Create Estimate)
* Tabs (Sale / Return / Exchange) should be an obvious segmented control under header.

---

## Component tasks (developer-level)

### 1) Header & app chrome

* Replace large white blank header with:

  * left: small logo, app title Point of Sale
  * center-left: segmented control for Sale/Return/Exchange
  * right: small buttons ‚Äî Dashboard (icon), Settings, Help
* Add subtle bottom border and drop shadow.

### 2) Search bar (primary interaction)

* Make full-width, centered, with large clickable area, placeholder: Search product name, SKU or scan barcode
* Add keyboard focus style (blue outline, subtle shadow).
* Inside search: left icon (magnifier), right: chip with live suggestions count, and scanner icon to open barcode scanner modal.
* Autocomplete dropdown:

  * show thumbnail, name, variant/SKU, price, stock, Add button.
  * keyboard support (‚Üë‚Üì, Enter to add).

*Acceptance:* Search should be focusable with F3 or / and scanner icon activates camera or scan modal.

### 3) Cart area (main)

* Two states:

  * Empty: large centered illustration + CTA Scan or search to add items with quick links (Recent items, Favorites).
  * Filled: list of cart item cards (compact) with:

    * Thumbnail, product name + SKU, unit price, qty selector (inline ¬±), discount per item, row total
    * Actions: edit (open modal), remove (trash), move to wishlist
  * Footer: subtotal, discount (global), tax, rounding, *Total (prominent)* and Pay button (primary green).

*UX details:*

* Make qty editable inline or by keyboard (select item and press +/-).
* Provide Hold Sale and Print buttons near Pay.
* Add Clear Cart as secondary, not primary ‚Äî confirm modal.

### 4) Right-side panel (Customer + Bill)

* Collapsible (can hide to maximize left area).
* Sections:

  * Customer information card (search/select/create customer)
  * Billing options (apply discount %, coupon code)
  * Payment method quick buttons (Cash, Card, UPI, Wallet)
  * Bill summary with clear rows and *big total* (green)
* Make fields optional and show placeholder microcopy.

### 5) Product search results panel (optional)

* When search returns many items, show grid with card layout and quick-add buttons.

### 6) Visual polish & icons

* Use consistent icon set (Heroicons, Feather or Material Icons).
* Use rounded avatars for product images.
* Soft hover effects, subtle transitions (ease 160ms).
* Use badges for low-stock or out-of-stock with clear colors.

---

## Accessibility & keyboard flow

* All controls keyboard-focusable, visible focus ring.
* Keyboard shortcuts:

  * / or F3 ‚Üí focus search
  * Ctrl+H ‚Üí Hold sale
  * Ctrl+C ‚Üí Clear cart (confirm)
  * P ‚Üí Open payment modal
* Support screen readers: meaningful aria-label, roles, and aria-live for server messages (e.g., "Item added to cart").
* Ensure color contrast ‚â• 4.5:1 for body text.

---

## Mobile / Tablet responsive rules

* Collapse right panel into a bottom sheet for tablet/phone.
* Use sticky header with search visible on small screens.
* Cart collapses into mini cart icon with item count when browsing.

---

## Micro-interactions & animations

* Add small animation when item is added to cart (fly-to-cart).
* Smooth slide for right-panel collapse.
* Toast notifications top-right for success/errors (auto dismiss 3s).

---

## Performance & loading

* Lazy-load product thumbnails.
* Debounce search input (300ms).
* Cache recent search results locally (sessionStorage).

---

## Acceptance criteria (what to hand back)

* Pixel-perfect responsive layout implemented with design tokens.
* Search, add-to-cart, qty-change, remove, clear cart ‚Äî all functional.
* Collapsible right panel works and states persist until logout.
* Keyboard shortcuts and accessibility roles implemented.
* Unit tests: basic UI tests (component render/interaction).
* Basic E2E flows (scan/add/pay simulation) in Playwright or Cypress.
* Deliverables: React components (or chosen framework) + Storybook entries + CSS tokens + screenshots for Desktop/Tablet/Mobile.

---

## Implementation suggestions / tech stack

* Frontend: React + TypeScript
* Styling: Tailwind CSS (or Chakra UI / Ant Design if preferred)
* State: Recoil/Redux/Context for cart
* Component previews: Storybook
* Testing: Jest + React Testing Library; E2E: Playwright/Cypress
* Icons: Heroicons
* Optional: use antd/Material for ready components but restyle to match tokens.
